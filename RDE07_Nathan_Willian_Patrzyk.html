Página Web e Website

Página Web

Arquivo onde se pode inserir textos, imagens, vídeos, entre outros. Possuem hipertextos que permitem a navegação entre páginas ou seções.

Websites ou Sites

Conjunto de páginas armazenadas em uma pasta ou subpasta relacionada de um servidor.

Arquitetura Cliente-Servidor e seus profissionais

Client-Side
Tecnologias: HTML, CSS, JavaScript.
Profissional: Front-End.
Server-Side

FullStack

HTML

Linguagem de Marcação de Hipertexto, baseada em tags e desenvolvida por Tim Berners-Lee

O HTML é usado na publicação de hipertexto, vinculados através de hyperlinks.


Tags

Realiza a marcação de conteúdo, possuem funcionalidades específicas, sem o seu uso é exibido um texto plano.

Código:
<tag> conteudo </tag>

Podem possuir atributos, que fornecem informações extras.

Código:
<tag atributo="valor"> conteudo </tag>

Hyperlinks

É usado como referência dentro de um documento para outras partes do mesmo, para outros documentos ou um endereço qualquer.
São divididos em:

  • Remotos
  • Internos
  • Locais
  • Com endereço ou arquivos

Marcação de Hyperlinks

A marcação hyperlinks é feita utilizando a tag <a> e a propriedade href que realiza a referência do conteudo

Sintaxe

<a href="endereço">Conteúdo Clicavel</a>

Hyperlinks Externos ou Remotos

São links que fazem referência a sites na web, no href usamos uma URL - Uniform Resource Locator para referenciar.

Sintaxe

<a href="URL">Conteúdo Clicavel</a>

Uso do target=_blank: adicionar no seu hyperlink, quando clicado o navegador abri-rá uma nova guia.

<a href="URL" target="_blank>Conteúdo Clicavel</a>

Uso do target="_self":Ao adicionar no seu hyperlink, quando clicado o navegador abri-rá uma nova guia.

<a href="URL" target="_self>Conteúdo Clicavel</a>

Uso do rel="external":Define o hyperlink como externo.

<a href="URL" target="_blank rel="external">Conteúdo Clicavel</a>

Uso do hreflang="Linguagem": Ao adicionar no seu hyperlink, quando clicado o navegador abri-rá uma nova guia.

<a href="URL" target="_blank rel="external" hreflang="Linguagem">Conteúdo Clicavel</a>

Hyperlinks Internos

São links que apontam para lugares diferentes em um mesmo documentos HTML.

Sintaxe

<a href="#referencia">Conteúdo Referenciado</a>

<a name="referencia">...</a>

Hyperlinks Locais

São links que apontam para paginas diferentes em HTML de um mesmo site.

Sintaxe

<a href="arquivo.html">Conteúdo Referenciado</a>

Hyperlinks em Imagens

São utilizados para criar hyperlink em uma imagem, utilizam a tag de imagem e a tag de hiperlink.

Sintaxe

<a href="referencia"><img src="imagem.jpg"></a>

Hyperlinks para Download de Arquivos

Apontam para arquivos e realizam o download dos arquivos selecionados.

Sintaxe

<a href="arquivo.extensão" download="arquivo.extensão" type="Media-Types"><Conteudo</a>

Media-Types - é um identificador padrão usado na Internet para indicar o tipo de dado que um arquivo contém. Media-Types

Hyperlinks para Endereço Eletrônico

Apontam para um endereço eletrônico dos arquivos selecionados.

Sintaxe

<a href="mailto:email@email.com"><email@email.com</a>

Softwares de Versionamento

Realiza o controle de versões de determinado projeto ou trabalho. Sendo ele de código ou documento, de uma pessoa ou de uma equipes de pessoas.
Funciona atravéz de um repositório remoto, que vai gerenciar e sincronizar tanto o histórico do repositório local como do repositório remoto.

Importante para evitar desencontros e problemas no trabalho em equipe.

Tipos de Versionamento

Centralizado
Repositório central em um serivdos com controle de versões, depende do servidor. Como exemplo a Software Subversion.
Distribuído
Repositório local com controle de versão, que são atualizadas no repositório central, independe do servidor. Como exemplo a Git

Git

Sofware livre de versionamento, criado para o Kernel do Linux por Linus Torvalds em 2005.

Sistema de versionamento distribuído.

GitHub

Plataforma de hospedagem de código-fonte e arquivos com controle de versão.

Preparo do Ambiente

  1. Baixar o Git: https://git-scm.com/
  2. Criar conta no GitHub: https://github.com/
  3. Bixar o GitHub Desktop: https://desktop.github.com/
  4. Abrir o VSCode: https://code.visualstudio.com/

Comandos GIT

commit
Torna permanente sua alterações do repositório local.
push
Envia as alterações para o servidor no repositório remoto.
fetch
Verifica se o repositório local está sicronizado com o servidor remoto
pull
Sincroniza o repositório local com o remoto atualizando as versões.

Crie seu Repositório

  1. No GitHub Desktop: Crie um novo repositório (Na licença altere par MIT License, para tornar seu repositório publico ou none para deixa-lo privado).
  2. No seu repositório: Publique o seu repositório público no GitHub (Ctrl+P --> Desmarque a opção Manter privado para manter o repositório público).
  3. Após criar e publicar seu repositório, abra-o no Visual Studio Code (Ctrl+Shift+A).
  4. Faça qualquer alteração no arquivo README e veja sua alteração no GitHub Desktop;
  5. No canto inferior do GitHub Desktop realize o commit, adicionando o título no Summary.
  6. Realize o push das suas alterações (Ctrl+P).
  7. No menu superior pode-se realizar o fetch para conferir se está devidadamente alterado.

Clonando Repositórios

  1. No GitHub: No menu incial, clique em New para criar um novo repositório.
  2. No GitHub Desktop: Abra sua lista de repositórios, clique em Add e Clone a repository selecione o repositório que deseja clonar e pressione Clone

Projetos Pré-Existentes

  1. No GitHub Desktop: Acesse File --> New repository (Ctrl+N).
  2. Ao criar o repositório: Selecione para iniciar com o README e escolha a Licença MIT
  3. Após copiar o conteúdo de seu projeto, realize o commit e publique seu repositório (Ctrl+P --> Desmarque a opção Manter privado para manter o repositório público)

Branches e Hospedagem

Branches

São ramificações de um projeto de software, tendo por objetivo o desenvolvimento paralelo de:

  1. Novas funcionalidades
  2. Correções de erros
  3. Alteração de código

Tipos de branches:

  • Branch Master (Main): Braço principal de um projeto em um repositório.
  • Branch Secundária: Braço secundário de um projeto em um repositório.

Passo a passo:

  • No GitHub Desktop crie um novo repositório.
  • Publique o seu repositório (Ctrl+P).
  • Abra seu repositório no Visula Studio e crie um arquivo index.html.
  • Edite o arquivo index.html como desejar.
